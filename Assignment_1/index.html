Paddle Practice

  • Originality: It's a game to hone those brickbreaker pong skills!
  • Technical Merit: Can't say anything worthy of interest.
  • Prototype Postmortem: How to use HTML a little bit, as well as code in JavaScript. Actually learning HTML, and trying my best to get it to actually work.
  • Prototype Assets: I did not make assets from scratch, instead I borrowed them. On the site below, I borrowed the assets from the Breakout game after downloading the zip file below the provided code.
    • Breakout Code: Breakout Code
  • Prototype Closest Other Game: The closest other game would be the game I borrowed, which is Breakout.
  • High Concept: A game to test how long can last and how high one can score playing pong by oneself with only one life.
  • Theme: None.
  • Mandated Variety: The input is meant to be for a mouse, and the genre arcade/casual. There is also no real randomness, and the playstyle is simple and reflex based.
  • Prototype Goal: Only game mechanic is trying to keep the ball from hitting the bottom of the screen by touching it with the paddle.
  • Player Experience Goals: A fun way to kill time, and increase your hand-eye coordination.
  • Gameplay: The player can move the paddle around with their mouse in order to hit the ball. Asides from that there are no other controls besides hitting enter to try again. The ball has physics based off that of the phaser engine, and the main goal is to last as long as one can before the ball hits the bottom of the screen.
  • Strategies: Being fast and nimble with a mouse/touchpad.
  • Story/Setting/Premise: There is no story for this game.
  • Target Audience: People bored on the internet looking for a game to play.
  • Play Time: As long as you are able to keep the ball on the field (the screen).
